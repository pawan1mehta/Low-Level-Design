# Singleton Design Pattern

## Overview
The **Singleton** is a **creational design pattern** that ensures a class has only **one instance** while providing a **global access point** to that instance.

## Problem it Solves
The Singleton pattern addresses two key problems:

1. **Controlled Access to a Shared Resource**  
   - Ensures that a resource (such as a **database connection** or a **file handler**) is accessed by only one instance.

2. **Global Access Point**  
   - Provides a **single, globally accessible** instance of a class, eliminating the need for multiple instantiations.
